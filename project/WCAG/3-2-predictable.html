3.2 Predictable

November 21st, 2016

Definition

“Make Web pages appear and operate in predictable ways” - World Wide Web Consortium.

What's that mean?

When a component of a website receives focus either from a mouse or a keyboard, it does not change the context of the content or function.

Why?

This criterion not only helps those who interact with content through a screen reader, but every other user as well. Ensuring Web pages operate in a predictable way improves the overall experience a user has. Additionally, this criterion can help alleviate frustration when a function operates in an entirely different way than expected.

Level AA

Though not necessary to meet the basic level of this criterion, consistency throughout your website is important for many of the same reasons. This is especially important when referring to global navigation. A navigation intended to be used on all pages should be the same and should not alter things such as order or links.

Example

An example of this criterion would be changing the context of a form a user is asked to fill out. If form fields appeared and hid depending on the user’s selection, the overall context is the same for the user. For instance, if a user selects the option to “Schedule A Meeting” different fields show than if a user were to select “Cancel a Meeting”. The form operates in a predictable way because a user would not expect or even need to see option to “Cancel a Meeting” when they are trying to instead schedule one.
