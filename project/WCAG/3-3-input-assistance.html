3.3 Input Assistance

November 28th, 2016

Definition

“Help users avoid and correct mistakes” - World Wide Web Consortium.

What's that mean?

Any error that could be shown throughout the interaction of your website should be made known to all users so they can identify the error that they made and then correct it.

Why?

Errors are fairly common when dealing with multiple varieties of input, and this criterion helps ensure users are still able to complete these tasks. By identifying errors, providing proper instruction, and allowing users to edit their input, they are allowed to complete the desired task and prevent the error the next time.

Example

This criterion is crucial when dealing with forms users must fill out in order to submit an email, sign up for an event, or any other purpose for a form. Because of the variances in form submission, such as the way dates are entered, or how a state is to be entered, forms often result in errors before they are properly submitted. If an error were to happen, it’s important that the user, whether they are submitting the form through a screen reader or not, is aware of the error and how they need to fix it. If this type of suggestion is not provided, the user might leave the site altogether with a poor experience.
