4.1 Compatible

December 5th, 2016

Definition

“Maximize compatibility with current and future user agents, including assistive technologies.” - World Wide Web Consortium.

What's that mean?

The criterion is in place to ensure content and Web elements are compatible with all technologies, while also ensuring they will be compatible in the future.

Why?

Much like Internet browsers, it’s important that all users can interact with your content. By creating content and developing Web elements in a semantic, industry-standard way, all users, no matter their assistive technologies, will be able to access the information in the same way.

Example

This criterion is best demonstrated through a demo of a working assistive technology and one that is broken because of the content. However, as I mentioned above, this is much like Internet browsers. Each browser has their own quirks and ways to display information, but to a web developer and to the creator of the browser itself, there are standards put in place to ensure widespread consistency. It is just as important for both the creator to build the browser to adhere to these rules, as much as it is important for the Web developer to create content that follows these rules. If you build solely for Google Chrome, your site may function similarly on Internet Explorer, but it won’t be the exact same. The same idea is applied to assistive technologies.
