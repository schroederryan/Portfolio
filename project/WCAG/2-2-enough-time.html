2.2 Enough Time

October 23rd, 2016

Definition

“Provide users enough time to read and use content” - World Wide Web Consortium.

What's that mean?

For every element on your website that is controlled by a time limit, you must provide alternatives for users to either turn off, adjust, or extend the time limit.

Why?

Because every user reads at a different pace and will interact with the content in a different way than someone else, it’s important to allow alternatives to time-controlled content. This can be done in a couple ways, such as allowing users the options to turn off the time limit altogether. It is also possible to allow for an extension. This ensures all users, interacting with content in a variety of ways, are able to read through and interact with all elements.

Exceptions

Though it is best to always provide an alternative to elements with time limits, there are a few situations where that is not possible.

  • Real Time: the time limit is essential for the event such as an auction.
  • Essential: if the time limit were removed, it would dramatically alter the activity.
  • 20 hours: if the time limit is longer than 20 hours, there is no need to provide alternatives to adjust or even extend the time.

Example

A great example for this criterion would be for an activity where you are asked to read a passage entirely within 5 minutes and then answer some questions about that passage. In this situation, it is not essential to the activity to have the 5 minute time limit, and all users might not be able to read the passage in that time. It might be best to simply allow the user to turn off the time limit and allow them to read at their pace and complete the activity to the best of their ability. Not every user to your website will be able to read at the same pace, which alienates those outside of the reading pace you set.
