3.1 Readable

November 14th, 2016

Definition

“Make text content readable and understandable” - World Wide Web Consortium.

What's that mean?

Though pretty straightforward, content throughout the website, on each page, should be understandable by all mediums like screen readers or any other assistive technology.

Why?

This criterion is included to ensure all content on a page or throughout a website can be understood by everyone. Screen readers do not yet have the ability to intelligently decipher different languages or acronyms throughout a website. Because of this limitation, users will not understand the full meaning of the content.

Example

An example of this criterion would be a webpage that has both English and Spanish words. This is common on webpages intended for students learning Spanish as a second language. In this situation, it would be important to specify the different languages programmatically to ensure universal understanding. If a user were to interact with the webpage through a screen reader, the use of language attributes would help them understand all of the content. A screen reader won’t be able to discern English from Spanish on its own, but can switch between the languages because they are programmatically specified.
